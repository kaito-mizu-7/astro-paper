正月休みの時間を使って[SQL 実践入門](https://amzn.asia/d/4kydXge)を読んでいました。<br>
普段の業務では、要件に沿った SQL はある程度自力で書けていましたが、そろそろパフォーマンスも意識したくなり、この本を購入しました。

#### レビュー

この本の良かったところは、SQL クエリのパフォーマンスを良くするための方法を DBMS の内部アーキテクチャの視点から 解説してくれている点です。
エンジニアがクエリを DBMS に投げてから、実際に実行されるまでの一連の流れを体系的に学べました。

実行計画の読み方、実行計画の安定性の重要性、インデックスの効率的な使い方、JOIN 時の結合アルゴリズムなど、どれも実務に直結する内容ばかりでした。

> SQL のパフォーマンスを決定する要因は、1 に I/O、2 に I/O、3 と 4 がなくて、5 に I/O

著者のミックさんは、とにかく「いかにテーブルの読み込み回数を減らすか」がパフォーマンスを左右する、という点を強く押していたのが印象的でした。

#### 最後に

かなり学びの多い一冊でした。
パフォーマンスの良いクエリを書くための基礎体力がついた気がします。

今後の業務では、実行計画を見てクエリを直して、また実行計画を確認して……という往復作業が増えそうですが、それも含めて楽しみです。
僕は Trino / Presto などの分散クエリエンジンをメインに業務で使っているので、分散環境特有の最適化について書かれた本や記事も、次は読んでみたいですね。<br>
